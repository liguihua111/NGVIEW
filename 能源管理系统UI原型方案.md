# 青州卷烟厂能源管理系统 UI 原型方案（基于 PC界面UI设计规范 V1.0）

## 1. 设计目标

面向卷烟厂日常能源管理业务，构建 **大气稳重、信息清晰、操作高效** 的 PC 端能源管理系统界面。  
本方案同时满足：

- 全中文界面；
- 继承《PC界面UI设计规范V1.0》的布局与组件规则；
- 兼容并统一帆软报表（FineReport）嵌入风格；
- 适配烟厂操作场景（车间、动力、管理层、能管人员、系统管理员）。

---

## 2. 设计依据

### 2.1 UI 规范关键约束（来自 V1.0）

- 主色：商务蓝 `#0091FF`（操作态、主按钮、可操作图标）。
- 字体：
  - Windows：`Microsoft YaHei`（中文）、`Arial`（英文）；
  - Mac：`PingFang SC`、`Helvetica`。
- 推荐字号：`14 / 16 / 24 / 32`（偶数阶梯，层级差 2~4px，行高约 1.5）。
- 布局与间距：
  - 基准间距：`10px`；
  - 顶部导航区高度：`64px`；
  - 标题区高度：`64px`；
  - 操作区高度：`64px`；
  - 左侧菜单收起宽度：`64px`；
  - 卡片圆角：`8px`；
  - 卡片上下间距：`20px`；
  - 树区与内容区间距：`16px`；
  - 通用帮助框最小安全边距：`160px`；
  - 页签超出时仅显示最近 `8` 个。

### 2.2 功能范围依据（青州能管功能 V3.3）

系统覆盖 13 大业务域：  
首页、能耗统计、能效分析、生产管理、能源平衡分析、预测模型、能源质量、能流图、综合管理、综合报表管理、数据录入、系统管理、一体化规划功能。

## 3. 整体信息架构（IA）

7. 能源质量  
   - 10类系统质量监测
8. 能流图  
   - 介质能流 + 全厂电/分变压器能流
9. 综合管理

    - 综合报表分析

## 4. 视觉风格定义（“大气 + 工业稳重”）

## 4.1 品牌风格

- 主风格：蓝灰工业风（稳重、专业、可靠）。
- 页面背景：浅灰底 + 白色卡片，突出数据与状态。
- 强调色：主蓝用于主操作，绿色/橙色/红色用于状态反馈（成功/预警/告警）。

### 4.2 色板建议（与 V1.0 兼容）

- 品牌主色：`#0091FF`
- 成功：`#00C96D`
- 警告：`#FF8400`
- 风险/告警：`#D7261D`
- 背景色：`#F5F7FA`
- 卡片底色：`#FFFFFF`
- 边框/分割线：`#E6E6E6`
- 主文本：`#1F2329`
- 次文本：`#4E5969`
- 弱文本：`#86909C`

> 说明：辅助色以规范为基线，可在图表中按“同饱和度不同色相”扩展，保证报表可读性。

### 4.3 工业场景可用性增强

- 关键按钮（查询、导出、提交）最小高度 `32px`，常用 `36~40px`；
- 告警信息支持颜色 + 图标 + 文案三级提示，避免只靠颜色识别；
- 查询条件优先单行展示，复杂筛选采用“展开/收起”；
- 高频页面支持“最近访问”和“车间快捷入口”（制丝/卷包/动力）。

## 6. 核心页面原型（低保真）

## 6.1 登录页（P01）

目标：品牌形象 + 安全登录 + 快速进入。

```text
┌──────────────────────────────────────────────────────┐
│ 顶部：青州卷烟厂能源管理系统                          │
│                                                      │
│ 中部左：品牌图（厂区/设备线稿）                      │
│ 中部右：登录卡片                                     │
│   [用户名]                                            │
│   [密码]                                              │
│   [验证码]                                            │
│   [记住账号]  [登录]                                  │
│                                                      │
│ 底部：版本号｜版权信息                                │
└──────────────────────────────────────────────────────┘
```

## 6.2 首页驾驶舱（P02）

核心模块：

- 今日综合能耗总览（电、水、蒸汽、天然气、压缩空气）
- 单耗与同比环比（按车间）
- 实时告警与处置状态
- 产量与能耗耦合趋势
- 设备运行健康度（空压/制冷/锅炉）
- 快捷入口（能耗统计、报警查询、手抄表录入、能源看板）

```text
┌ KPI卡1 ┬ KPI卡2 ┬ KPI卡3 ┬ KPI卡4 ┬ KPI卡5 ┐
├──────────────趋势图：全厂综合能耗(24h)──────────────┤
├────车间对标排行────┬────告警待处理────┬──设备健康──┤
└────────────────────────────────────────────────────┘
```

## 6.3 能耗统计（P03）

- 查询条件：时间、区域（生产/非生产）、介质、部门、班次；
- 内容：趋势图 + 分项占比 + 明细表；
- 支持按介质切换和导出。

## 6.4 能效分析（P04）

- 维度：全厂/部门/系统/介质；
- 重点：单位产量能耗、偏差分析、TOP异常点；
- 提供“分析结论卡片”与“建议操作”。

## 6.5 生产管理（P05）

- 产量管理：卷包、制丝、总产量；
- 碳管理：碳排放、折标煤；
- 卓越绩效：车间考核看板、评分明细、趋势追踪。

## 6.6 能源平衡分析（P06）

- 支持水平衡/电平衡/蒸汽/压缩空气平衡图；
- 左侧树：系统/回路/分支；
- 右侧卡片：输入-输出-损耗闭环校核。

## 6.7 预测模型（P07）

- 能源预测：日/月/年；
- 智能调度建议：空调、制冷、锅炉、空压、真空；
- 预测 vs 实际偏差曲线 + 模型可信度标识。

## 6.8 能源质量（P08）

- 10 类系统质量指标（压力、温度、流量、稳定度、合格率等）；
- 告警阈值标识 + 趋势诊断 + 异常时间段定位。

## 6.9 能流图（P09）

- 介质流向可视化（蒸汽/天然气/压缩空气/电等）；
- 支持钻取到变压器、支路、车间节点；
- 默认展示“全厂总览”，可一键切换“区域视图”。

## 6.10 综合管理（P10）

- 报警管理：预警查询、报警查询、闭环处置；
- 通知公告：个人消息、公告发布；
- 文件管理：制度、标准、操作文档归档。

## 6.11 综合报表管理（P11，含帆软报表）

- 综合报表分析
- 成本核算查询
- 能源看板

页面结构建议：

```text
标题区 + 操作区
┌ 查询参数区（系统原生样式） ┐
├ 工具栏（导出/打印/订阅）   ┤
└ 帆软报表容器（iframe/嵌入） ┘
```

## 6.12 数据录入（P12）

- 系数设置、公式设置、预警点、设备信息；
- 手抄表录入：水/天然气/汽油/外购二氧化碳/其他；
- 录入页优先三列栅格，必填项统一红色星号提示。

## 6.13 系统管理（P13）

- 组织、角色、用户、菜单、字典、日志；
- 权限颗粒度：菜单级 + 按钮级 + 数据域级；
- 审计日志可追溯（操作人、时间、IP、前后值）。

## 7. 帆软报表（FineReport）风格统一方案

## 7.1 统一原则

1. 报表外壳使用系统卡片容器（圆角 8px、边框 E6E6E6）。
2. 报表内字体统一为 `Microsoft YaHei`，字号 14/16。
3. 报表按钮风格与系统主按钮一致（主蓝、hover、disabled）。
4. 报表查询参数区与系统查询区一致（控件高度、间距、标签样式一致）。
5. 图表色板与系统色板统一，避免“系统蓝灰 + 报表荧光”割裂。

## 7.2 帆软页面组件映射

| 系统规范组件 | 帆软对应区域 | 统一规则 |
|---|---|---|
| 标题区 | 报表标题 | 左对齐 24px/16px 两级标题 |
| 操作区 | 工具栏（导出/打印） | 按钮高度与间距一致 |
| 查询区 | 参数面板 | 单行优先，超出可折叠 |
| 内容卡片 | 报表主体 | 白底卡片、圆角8px、阴影弱化 |
| 表格样式 | 数据集表格 | 表头浅灰、行高统一、斑马纹可选 |

## 7.3 报表模板建议（至少）

1. **全厂能耗日报**：介质分项 + 分时曲线 + 同比环比；
2. **车间单耗对标报表**：卷包/制丝/动力横向对比；
3. **成本核算月报**：费用构成 + 预算偏差 + 排名；
4. **碳排放与折标煤报表**：趋势 + 结构 + 归因；
5. **告警闭环报表**：告警量、平均处置时长、超时统计。

## 7.4 嵌入交互要求

- 报表加载时显示系统统一骨架屏；
- 报表异常时显示系统统一空状态/错误提示；
- 支持“从系统筛选条件透传到帆软参数”；
- 支持报表全屏、导出、打印，不破坏系统导航框架。

## 8. 关键业务流程原型

## 8.1 能耗异常处理流程

首页告警卡片 → 报警查询 → 查看点位趋势 → 下发处置通知 → 处置反馈 → 闭环归档

## 8.2 预测调度流程

预测模型页选择系统与时间范围 → 生成预测 → 对比实际偏差 → 生成调度建议 → 发布执行

## 8.3 报表分析流程

综合报表管理 → 选择报表模板 → 设置参数 → 查看/钻取 → 导出（Excel/PDF）→ 留痕
